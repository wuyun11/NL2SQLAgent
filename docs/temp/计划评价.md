## 这份计划做对了什么

### 1. 范围控制是对的

计划明确 Phase 1 只做：

```text

hello graph test

并且明确禁止：

```

这个边界非常好。

这正好对应 Phase 1 的目标：只验证 LangGraph 运行底座，不提前进入业务。原设计文档里也强调 Phase 1 不做 NL2SQL，只解决 `thread_id`、checkpointer、`graph.invoke`、`graph.stream`、hello graph 验证通道。

---

### 2. checkpointer 的位置改对了

计划现在写的是：

```text
platform/persistence creates checkpointers
bootstrap exposes the checkpointer and GraphRuntime
GraphRuntime only builds RunnableConfig and calls compiled graph invoke / stream
```

这个我支持。

LangGraph 官方文档里，checkpointer 是在 `graph.compile(checkpointer=...)` 时传入的；编译后，graph 会在每一步保存 state checkpoint，并且这些 checkpoint 按 thread 组织。([LangChain 文档][1]) 所以 checkpointer 更像是**graph 编译依赖**，不是 `GraphRuntime.invoke()` 的内部成员。

计划里这样分工是清楚的：

```text
app.checkpointer
  给 build_hello_graph(checkpointer=...) / 未来 build_nl2sql_graph(...) 用

app.graph_runtime
  只负责 invoke / stream / config 构造
```

这比 `app.graph_runtime.checkpointer` 更干净。

---

### 3. thread_id 设计正确

计划里 `resolve_thread_id` 的规则很清楚：

```text
显式 thread_id 优先
显式 thread_id 会 strip
空 thread_id 回退为 thread-{run_id}
未传 thread_id 回退为 thread-{run_id}
空 run_id 抛 ValueError
```

这很合理。

LangGraph persistence 要求调用有 checkpointer 的 graph 时，在 config 的 `configurable` 里提供 `thread_id`。([LangChain 文档][1]) 所以 Phase 1 即使只用 memory checkpointer，也坚持通过 `thread_id` 调用 graph，这一点是正确的。

---

### 4. GraphRuntime 的职责清楚

计划里的 `GraphRuntime` 只做：

```text
构造 RunnableConfig
注入 thread_id
注入 run_id / run_date metadata
调用 graph.invoke
调用 graph.stream
```

并且 `stream()` 原样返回 `list(graph.stream(...))`。

这正好符合 Phase 1 的克制原则。LangGraph streaming 文档里，`updates` 和 `values` 的返回语义不同：`updates` 返回每步 state update，`values` 返回每步完整 state。([LangChain 文档][2]) 所以现在不要急着封装成自己的业务事件，原样返回是对的。

---

### 5. hello graph 只作为测试工具，边界正确

计划没有新增 CLI 命令，而是只在测试中构建 hello graph。这个很好。

原设计文档也明确说：hello graph 是运行底座测试工具，不是用户功能；Phase 1 推荐只写测试，不扩 CLI。

这样不会污染 Phase 0 的 CLI startup，也不会把 hello graph 变成 app 的一部分。

## 建议执行前微调的 2 个点

### 1. Task 1 里缺少“安装依赖”的具体命令

计划写了：

```text
If this fails because langgraph is not installed, install project dependencies in the local environment before continuing.
```

这个方向没问题，但给 AI 执行时最好明确命令。否则它可能乱用 `pip install langgraph`，绕开你的 `pyproject.toml`。

建议补成：

```powershell
$python = (Get-Content -Raw .\.ai\local\python_path.txt).Trim()
& $python -m pip install -e ".[dev]"
```

或者如果你不想 editable install：

```powershell
& $python -m pip install -e .
```

更推荐第一种，因为后面要跑 pytest。

---

### 2. Task 5 的 stream updates 测试可以稍微更具体

现在是：

```python
assert chunks
assert any("greet" in chunk for chunk in chunks if isinstance(chunk, dict))
```

这个可以跑，但断言偏弱。更稳一点可以写：

```python
assert chunks
assert any(
    isinstance(chunk, dict)
    and "greet" in chunk
    and chunk["greet"].get("message") == "hello, Bob"
    for chunk in chunks
)
```

因为你是想验证 `updates` 里确实有 `greet` 节点的 update，而不只是出现了一个 key 叫 `greet`。

不过这个不是阻塞项。
