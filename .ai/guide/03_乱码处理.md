# Windows 终端与搜索命令处理

## Description

说明在当前这台 Windows 电脑、PowerShell / Codex 场景下，优先使用本项目自带搜索工具，并处理中文乱码问题。

## Trigger

以下情况优先参考本文件：

- 搜索文件、搜索文本
- 读取中文文件
- 输出中文内容
- 写入中文文本
- 处理 Markdown、配置、文档
- 发现文件在编辑器里正常、在终端里异常时

## Instructions

### 搜索命令优先级

- 在本仓库内搜索文件或文本时，优先使用本项目自带的：

```powershell
.\.ai\local\tools\rg.exe
```

- 不要默认使用系统 PATH 里的 `rg`
- 不要默认使用 Codex 或其他工具自带的 `rg`
- 如果 `.ai/local/tools/rg.exe` 不存在，向用户说明是否要先处理rg工具不存在的问题

常用写法：

```powershell
.\.ai\local\tools\rg.exe --files
.\.ai\local\tools\rg.exe "keyword"
```

### 默认编码处理

- 优先先在命令里显式设置 UTF-8：

```powershell
[Console]::InputEncoding = [System.Text.Encoding]::UTF8
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
$OutputEncoding = [System.Text.Encoding]::UTF8
chcp 65001 > $null
```

- 然后再执行实际命令
- 已知当前环境里：
  - 即使 `chcp` 是 `65001`
  - 即使 `Console.InputEncoding` 和 `Console.OutputEncoding` 显示为 `utf-8`
  - `$OutputEncoding` 仍可能默认是 `us-ascii`
- 因此不能只看 code page，也不能只改 `Console.OutputEncoding`

### 文件读写

- 读取文本文件时，优先显式指定编码：

```powershell
Get-Content .\README.md -Encoding UTF8
```

- 写入文本文件时，优先显式指定编码：

```powershell
Set-Content .\note.md $text -Encoding UTF8
```

### 排查顺序

- 如果仍然乱码，按下面顺序排查：
  1. 是否已经显式设置 UTF-8
  2. 是否显式指定了 `-Encoding UTF8`
  3. 是否漏掉了 `$OutputEncoding`
  4. 是否是终端显示问题，而不是文件本身损坏
